An advanced educational hybrid cryptographic suite built entirely on an Android environment using **Pydroid 3** and **Termux**. SSE-U1 combines symmetric and asymmetric cryptography to protect local files with absolute integrity.

---

## 🚀 Key Features
* 🛡️ **Argon2id KDF:** High-grade memory-hard password hashing.
* 🔑 **AES-256-CBC:** Enterprise-grade symmetric file encryption.
* 🔍 **HMAC-SHA256:** Strict cryptographic sealing to prevent data tampering.
* 📦 **RSA-2048 (PKCS1_OAEP):** Asymmetric key encapsulation for secure session key transport.
* 💾 **Key Persistence:** Persistent `.pem` storage architecture for automated key loading.

---

## 🛠️ Mathematical & Logical Architecture

The software structure channels data through secured pipelines:

| Phase | Operation | Primitive Used | Output Form |
| :--- | :--- | :--- | :--- |
| **1** | Key Derivation | Argon2id (64-byte split) | 32B AES Key / 32B HMAC Key |
| **2** | Session Protection| RSA-2048 PKCS1_OAEP | 256-byte Protected Block |
| **3** | Data Encryption | AES-256-CBC + Random IV | Padded Ciphertext |
| **4** | Integrity Sealing | HMAC-SHA256 | 32-byte Authentication Tag |

---

## 💻 Custom Binary Package Specification (.sseu1)

Every protected package generated by the core engine compiles down into a strict custom-packed binary file layout:

> **[ 256 Bytes: Protected AES Key ] → [ 16 Bytes: Salt ] → [ 16 Bytes: IV ] → [ 32 Bytes: HMAC Tag ] → [ Variable: Ciphertext Space ]**
